# GMDSOFT Design System v2.12

## Visual Identity

GMDSOFT builds industrial-grade Windows desktop applications for video production, security, and device management. The design philosophy is **precision engineering for professionals**. Every interface element maximizes operator productivity in high-stakes environments.

- Platform: Windows desktop application
- Resolution: 1920x1080 FHD (min 1280x720)
- Density: Information-dense, optimized for 8-hour operator shifts
- Modes: Light (default) / Dark (low-light monitoring)
- Seed Color: `#4681DB` — generates the entire color system

## App Shell Structure

```
+-----------------------------------------------------+
|  Windows Title Bar (H: 32px)              _ [] X     |
+----------+------------------------------------------+
|          |                                          |
|  Side    |  Content Area                            |
|  Nav     |  (padding: 24~32px)                      |
|          |                                          |
|  W: 240  |  W: Fill (min 1040px)                    |
|  (expand)|                                          |
|  W: 56   |                                          |
| (collapse)|                                         |
+----------+------------------------------------------+
```

- Title Bar: 32px, Windows-native chrome
- Side Navigation: 240px expanded / 56px collapsed (icon-only)
- Content Area: fluid width, min 1040px

## Absolute Rules

1. **Font sizes**: Only 10 / 12 / **14** / 16 / 18 / 20 / 22 / 28px. Default is 14px. No 13px, 15px, 24px.
2. **Font weights**: Only 400 (Regular) and 700 (Bold). No light, medium, semibold.
3. **Letter-spacing**: Never positive (> 0px). Only 0 or negative values allowed.
4. **Spacing**: Only 4px multiples (0, 2, 4, 8, 12, 16, 20, 24, 32, 40, 48, 56, 64, 80, 96).
5. **Colors**: Only token-defined colors. No arbitrary HEX values. Specify as `token-path #HEX` pair.
6. **States**: Every interactive element must define base / hover / pressed / disabled.
7. **Accessibility**: Never use color alone to indicate status — always pair with text. Never use icons alone — pair with labels.
8. **No emojis**: Use monoline SVG icons (16/24/32px) instead.
9. **No gradients**: Solid token colors only for backgrounds and buttons.
10. **No decorative elements**: No textures, patterns, or illustrations.

## Dark Mode Key Differences

In Dark mode, switch `<html data-theme="dark">`:
- Page bg: `#141416`, Card bg: `#252529`, Section: `#1E1E21`
- Primary text: `#E0E0E0`, Secondary: `#A0A6AC`
- Primary: `#4F94D4`, Seed: `#75A3E8`
- Borders lighten: `#3A3C42`, Dividers: `#2A2C30`
- Scrim deepens: `rgba(0,0,0,0.8)`
- Depth via surface stepping: `#141416` → `#1E1E21` → `#252529` → `#2D2D30`
