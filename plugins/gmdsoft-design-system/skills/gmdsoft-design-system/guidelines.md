# 적용 가이드라인 (Guidelines)

> 출처: index.html 가이드라인 섹션(`#i18n-truncation`). 컴포넌트를 실제 화면에 조립할 때 따르는 운영 규칙.

## 다국어 잘림 (Truncation)

같은 의미라도 언어에 따라 길이가 다르다. 독일어·러시아어 등에서는 짧은 단어가 2~3배까지 길어진다. 모든 잘림을 막을 수는 없으므로 기준은 하나다 — 잘리더라도 전체 내용을 확인할 방법이 항상 있어야 한다.

### 불변 조건 (Invariants)
1. **클리핑 금지** — 글자가 중간에 뚝 끊기면 안 된다. 잘림은 항상 말줄임표(…)로 표시.
2. **전문 확인 수단** — 잘린 텍스트는 마우스 오버 툴팁으로 전체 내용을 확인할 수 있어야 한다.
3. **주요 액션은 잘리지 않는다** — 실행·저장·삭제 같은 주요 버튼 라벨은 어떤 지원 해상도에서도 온전히 보여야 한다.

- 기본 화면(1280×720)과 패널 기본 크기에서는 잘림 없이 표시한다. 사용자가 그보다 줄이면 위 세 조건만 보장한다.
- 잘린 글자는 최소 4자가 남아야 한다.
- 페이지 제목·오류 메시지·알림은 자르지 않는다.

### 언어 전환 / 설계 기준
- 기본 표시·신규 화면 폭 산정은 **독일어 기준**. 지원 언어 중 가장 길어지는 언어라 잘림을 바로 확인할 수 있다.
- 지원 언어: 한국어(ko)·English(en)·Deutsch(de)·Русский(ru)·Tiếng Việt(vi).

### 케이스별 처리 정책
- **A. 폭 조절 가능 컬럼 — 잘림 허용**: 사용자가 폭을 직접 조절하는 표는 잘림 허용. 단 말줄임표 + 툴팁 제공, 초기 폭은 제목이 온전히 보이게 정한다.
- **B. 고정 폭 컨트롤 — 잘림 금지**: 버튼·탭처럼 사용자가 폭을 못 바꾸는 컨트롤은 자르지 않는다. 폭은 글자 길이에 맞춰 늘어나되 max-width를 넘지 않는다. 최대 폭에 닿으면 더 짧은 번역으로 바꾸고, 그래도 어려울 때만 말줄임표를 쓴다. 실행·저장 등 주요 버튼은 어떤 경우에도 자르지 않는다.
- **C. 폭 조절 불가한 표 — 말줄임 + 툴팁**: 말줄임표 + 툴팁으로 처리하고, 더 짧은 번역을 함께 검토한다. 용어 축약 시 Confluence 공식 표기를 따른다. 신규 화면은 처음부터 폭 조절 가능한 표로 만든다.
- **D. 안내 문구 — 줄바꿈 (Wrap)**: 안내 문구는 자르지 않고 줄을 바꾼다. 영역은 아래로 늘어나고, 버튼과의 간격은 **8px 이상**을 지킨다.
- **E. 사용자 데이터 — 가운데 말줄임**: 파일명 등 번역되지 않는 사용자 데이터는 항상 말줄임표 + 툴팁을 쓰고, 확장자가 보이도록 **가운데를 줄인다**.

### 폭 규칙 (Width)
> 값은 tokens.css의 `--gmd-width-*` 토큰. 최소 폭보다 좁아지면 알아볼 수 없고, 최대 폭에 닿으면 폭을 더 늘리는 대신 더 짧은 번역으로 바꾼다. (Design Kit 반영 검토 대상)

| 대상 | min-width | max-width | 근거 |
|---|---|---|---|
| Button (md 이상) | `--gmd-width-button-min` 88px | `--gmd-width-button-max` 240px | 업계 표준 최소 폭(Material) · 긴 번역 대비 상한 |
| Button (xm·sm·다이얼로그) | `--gmd-width-button-min-compact` 64px | 240px 동일 | 다이얼로그·조밀 영역 표준(Material) |
| Text Link 버튼 | 없음 (콘텐츠 폭) | 240px 동일 | 문장 속 버튼이라 최소 폭 없음 |
| 표 컬럼 | `--gmd-width-column-min` 48px | 없음 (사용자 조절) | 말줄임표 + 최소 4자 보이는 폭(IBM) |
| Input · TextBox | 그룹 폭에 맞춰 확장 | `--gmd-width-field-max` 320px | 같은 화면 입력 폭 통일(Windows 권고) |
| 클릭 타깃 공통 | `--gmd-width-target-min` 24px | — | 접근성 기준(WCAG) 최소 클릭 크기 |

- 최대 폭에 닿은 버튼은 글자를 자르지 않고 더 짧은 번역으로 바꾼다.
- 폭을 늘릴 때는 라벨이 긴 그 버튼만 늘리고, 옆 버튼을 따라 넓히지 않는다.

### 처리 수단 우선순위
1. **레이아웃으로 해결** — 폭을 늘리거나 줄을 바꾼다. 텍스트를 건드리지 않는 가장 안전한 방법.
2. **다국어 리소스 축약** — 더 짧은 표현으로 바꾼다. 용어는 Confluence 공식 표기를 따른다.
3. **말줄임 + 툴팁** — 위 두 가지가 어려울 때 쓰는 마지막 표시 방법. 주요 버튼에는 쓰지 않는다.
4. **컨트롤 구조 변경** — 비용이 가장 큰 수단. 효과를 따로 판단해 결정한다.

### 근거
- W3C 텍스트 팽창: 10자 이하 200~300%, 70자 이상도 130%까지 길어짐.
- IBM Carbon: 잘린 항목은 항상 툴팁, 최소 4자 유지, 제목·오류 메시지 잘림 금지.
- Microsoft 현지화 가이드: 텍스트 영역 +30%, 짧은 텍스트는 최대 +200% 여유.
- Material Design: 버튼 최소 폭 88dp, 다이얼로그 64dp.
- WCAG 2.2: 클릭 영역 최소 24×24px.
- Windows 데스크톱 가이드: 기본 버튼 폭 사용, 라벨이 긴 버튼만 확장.
- 사내 사례: REQ-516, IDEA-571.
